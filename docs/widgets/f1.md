# F1 Widget

Displays the next Formula 1 race and the current top-5 driver standings. No configuration required.

## Storage Keys

| Key | Value |
|---|---|
| `f1-cache-{YYYY-MM-DDTHH}` | Hourly JSON cache of `{ race, standings }` |

The cache key includes the hour (`toISOString().slice(0, 13)`) so cached data is reused for the rest of the current hour and then discarded on the next load.

## Data Source

`GET /api/f1` fetches from the [Jolpica F1 API](https://jolpi.ca) (the maintained Ergast replacement) in parallel:

- `https://api.jolpi.ca/ergast/f1/current/next.json` — next scheduled race
- `https://api.jolpi.ca/ergast/f1/current/driverStandings.json` — current standings

Both requests use Next.js `revalidate: 3600` so the server-side cache also refreshes hourly.

The route returns:

```ts
{
  race: {
    raceName: string;
    round: string;
    date: string;       // "YYYY-MM-DD"
    time?: string;      // "HH:MM:SSZ"
    Circuit: {
      circuitId: string;
      circuitName: string;
      Location: { country: string };
    };
  } | null;
  standings: DriverStanding[];  // top 5
}
```

`race` is `null` when the season is complete and no next race exists.

## Circuit SVGs

Circuit outline SVGs are served from `/circuits/{circuitId}.svg`. The `circuitId` value comes directly from Jolpica and matches the filenames in `public/circuits/` — no mapping table is needed. SVGs are from [Jules Roy's f1-circuits-svg collection](https://github.com/julesr0y/f1-circuits-svg) (CC BY 4.0).

The `<img>` is rendered at `w-16 h-16` with `opacity-20` (light mode) / `opacity-25 invert` (dark mode) so it sits as a subtle background element next to the race info.

## Countdown

`countdown(dateStr, timeStr)` computes the diff between now and the race datetime. It returns `"race day"` if the race is today or past, `"tomorrow"` if exactly 1 day away, or `"in N days"` otherwise.

## Standings Display

The top 5 standings show position, three-letter driver code, family name, and points. All in a compact table-like layout with `tabular-nums` for aligned numbers.

## Scroll Fades

Top and bottom gradient fades are tracked via `ResizeObserver` and scroll events.
